Spring 框架提供了利用JdbcTemplate 直接访问JDBC从而操作数据库的技术。另外可以利用Hibernate或Spring Data提供的Repository生成查询

嵌入式数据库

Spring Boot 提供了嵌入式的H2、HSQL和Dery数据库。无需提供连接URL，只需要包含对应的嵌入式数据库的依赖项。例如：

如果类路径上有多个嵌入式数据库，可以配置spring.datasource.embedded-database-connection属性设置使用哪一个。将其设置为none禁用嵌入式数据库的自动配置。

在测试环境下，多个应用程序商家问会重用一个数据库，如果要确保每个上下文都有单独的嵌入式数据库，需要配置

如果为嵌入式数据库配置了URL，应确保禁用数据库的自动关闭，这样可以让spring boot控制何时关闭数据库，从而确保只使用一次数据库之后就不再须要的情况。

  • spring.datasource.hikari.*；
  • spring.datasource.tomcat.*；

连接池

spring boot选择连接池的规则：

  1. 如果HikariCP 可用，则选择；
  2. 否则，如果 Tomcat 池DataSource可用，则使用；
  3. 否则，如果Commons DBCP2可用，则使用；
  4. 如果上面都不可用，并且Oracle UCP可用，则使用；

如果加入了spring-boot-starter-jdbc或spring-boot-starter-data-jpa的依赖会自动选择HikariCP。如果不希望使用上述规则可以如下设置：

使用DataSourceBuilder可以通过手动配置额外的连接池.如果自定义 DataSource ，那么自动配置不会剩下。DataSourceBuilder支持以下连接池

  • HikariCP
  • Tomcat pooling Datasource
  • Commons DBCP2
  • Oracle UCP & OracleDataSource
  • Spring Framework’s SimpleDriverDataSource
  • H2 JdbcDataSource
  • PostgreSQL PGSimpleDataSource

JdbcTemplate
