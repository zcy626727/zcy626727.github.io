Controller

控制器（controller）用来处理请求，然后返回模板或Json数据，Phoenix 控制器也是基于 Plug 软件包构建的，并且本身就是Plug。

声明 Controller

路由的示例如下：

  • conn：连接结构体

  • params：接收传递的参数，建议以模式匹配的方式接收数据

命名规范

  • index：呈现给定资源类型的所有项目的列表
  • show：按 ID 呈现单个项目
  • new：呈现用于创建新项目的表单
  • create：接收一项新项目的参数并将其保存在数据存储中
  • edit：按 ID 检索单个项目并将其显示在表单中以供编辑
  • update：接收一个已编辑项目的参数并将该项目保存到数据存储中
  • delete：接收要删除的项目的 ID 并将其从数据存储中删除

渲染

控制器可以以多种方式渲染内容：

  • 纯文本：text(conn, "From messenger #{messenger}")
  • Json：json(conn, %{id: messenger})
  • HTML：render(conn, :show, messenger: messenger)

模板传惨

如果希望在使用render是传递参数，有两种方式：

  • 使用关键字传递
  • 使用Plug.Conn.assign/3方法：该方法返回一个conn
