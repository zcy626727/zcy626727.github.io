Kubernetes 配置及命令

使用 kubernetes 时主要通过命令和配置文件的方式描述核心组件操作 API 对象的目标状态，本篇介绍核心技术中各个组件的不同配置方式

kubectl工具支持三种对象管理方式：

  • 命令式对象管理：使用一条条指令操作对象
  • 命令式对象配置：通过命令配置和配置文件操作对象
  • 声明式对象配置：通过apply命令和配置文件操作对象

命令式对象管理

使用kubectl命令行工具提供的指令对对象进行管理，语法格式如下：

  • command：执行的操作
  • type：对象的类型
  • name：指定对象的名称，大小写敏感
  • flags：额外的参数

说明：

  • 为节省位置，后文列出的命令都省略了kubectl。

命令式对象配置

命令式对象配置是使用配置文件和命令想结合的方式管理对象。操作步骤如下：

  1. 首先编写配置文件，配置文件内包含对对象的声明和约束
  2. 使用命令执行配置文件进而操作对象

常用命令

命令 命令作用
create -f <filename|url> 创建对象
replace -f <filename|url> 更新对象
delete -f <filename|url> 删除对象
get -f <filename|url> -o yaml 查看对象

声明式对象配置

声明式对象配置是将所有对于对象的描述都声明在配置文件中，只需要执行apply命令执行配置文件目录，Kubernetes会创建指定目录中配置文件所定义的所有对象除非对应对象已经存在：

常用命令

命令 命令作用
apply -f <目录/> 创建/更新对象，添加-R可以递归地处理目录
diff -f <配置文件|目录/> 打印出将被创建的对象
get -f <配置文件> 打印现时配置
delete -f <文件名> 删除对象

在使用apply命令后也可以手动使用命令来更新集群的配置，比如使用scale进行扩容时，当前最新的配置就叫现时配置。
